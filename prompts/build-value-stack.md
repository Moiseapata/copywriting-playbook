# Build Value Stack Prompt

**Tags:** #prompt #offer-construction #value #stacking #claude

## How to Use This Prompt

When you need to build or enhance an offer by stacking value until the price feels like a fraction of the whole.

---

## The Prompt
Help me build a value stack for this offer:

Core Product: [describe]
Current Price: [$]
Target Customer: [describe]

Use Russell Brunson's Offer Stacking framework:

Step 1: List the Core Components

What's included in the base offer?

Step 2: The "Next Problem" Cascade

For each component, ask: "Once they have this, what's the NEXT problem they'll face?"

Core product → creates problem A → solve with Bonus 1
Bonus 1 → creates problem B → solve with Bonus 2
Bonus 2 → creates problem C → solve with Bonus 3
Continue until the offer feels comprehensive.
Step 3: Assign Realistic Values

What would each component cost if purchased separately? Use real market prices, not inflated numbers.

Step 4: The Stack

Present it as:

Component A ($X value)
Component B ($Y value)
Component C ($Z value)
Total Value: $XXX
Their Price: $Y
Step 5: The "Gap" Principle

The gap between total value and price should be significant enough that the price feels like a fraction of the whole.

Step 6: The "No-Brainer" Test

After seeing the stack, would a reasonable person feel stupid saying no? If not, add more value or lower the price.

Build the stack now.


---

## The Framework This Is Based On

**Offer Stacking (Brunson):** Every solution creates a new problem. The best offers anticipate and solve the cascade. A complete offer is not a product — it's a kit: the map, the Sherpa, and the gear.
